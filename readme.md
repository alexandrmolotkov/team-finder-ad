# TeamFinder — платформа для поиска команд для pet-проектов

TeamFinder — это веб-приложение, созданное для объединения разработчиков, дизайнеров и других IT-специалистов. Здесь можно найти единомышленников для совместной работы над pet-проектами, опубликовать свою идею, собрать команду и управлять проектами.

---

## 🚀 Функциональные возможности

- **Регистрация и авторизация** — создание аккаунта по email и паролю, вход в систему и выход.
- **Управление проектами** — создание, редактирование и завершение собственных проектов.
- **Участие в проектах** — возможность присоединиться к проектам других пользователей.
- **Избранное** — добавление понравившихся проектов в личный список избранного.
- **Фильтрация пользователей** — поиск участников по четырём критериям: авторы избранных проектов, участники моих проектов, авторы проектов, в которых я участвую, и пользователи, которым нравятся мои проекты.
- **Профили пользователей** — публичные страницы с контактами, списком навыков и проектов.
- **Редактирование профиля и смена пароля** — персонализация аккаунта и управление безопасностью.
- **Административная панель** — управление пользователями и проектами через стандартную админку Django.

## 🛠 Используемые технологии

- **Язык:** Python 3.10+
- **Фреймворк:** Django 4.x
- **База данных:** PostgreSQL
- **Фронтенд:** HTML, CSS, JavaScript (встроенные шаблоны)

## 📦 Установка и запуск проекта

### 1. Клонирование репозитория
```bash
git clone <ссылка_на_ваш_репозиторий>
cd team-finder-ad-main-3
```

### 2. Создание виртуального окружения
```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Настройка базы данных (PostgreSQL)

Установите PostgreSQL локально и создайте в pgAdmin:
 - **Базу данных с именем team_finder**
 - **Пользователя team_finder с паролем team_finder**
 - **Выдайте пользователю все права на базу данных**
Либо используйте Docker (если поддерживается вашей системой):
```bash
docker-compose up -d
```

### 5. Миграции
```bash
python manage.py makemigrations
python manage.py migrate
```

### 6. Создание суперпользователя
```bash
python manage.py createsuperuser
```

### 7. Запуск сервера
```bash
python manage.py runserver
```

### 8. Открытие приложения
Перейдите в браузере по адресу: http://127.0.0.1:8000/

## 🔐 Доступ к админ-панели

 - **Адрес:** http://127.0.0.1:8000/admin/
 - **Вход:** email и пароль суперпользователя, созданного на шаге 7.
